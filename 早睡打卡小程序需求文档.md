## 1. 产品概述

### 1.1 产品名称

**早睡星人** — 群友早睡打卡小程序

### 1.2 产品定位

一款基于微信群的社交化自律工具，通过"押金 + 打卡 + 排行榜 + 奖励"的闭环机制，帮助群友克服熬夜习惯，养成早睡生活方式。产品本质是**用社交压力和经济激励双重驱动行为改变**。

### 1.3 核心价值主张

| 价值维度 | 描述 |
|----------|------|
| **社交驱动** | 群友互相监督，熬夜会被看见，早睡会被认可 |
| **经济+荣誉双激励** | 罚金池充裕时经济奖励驱动，罚金池不足时称号/勋章/群表彰等社交荣誉补位，确保任何全勤率下坚持早睡都有回报 |
| **游戏化体验** | 早睡币、排行榜、称号系统让自律变得有趣 |
| **正向循环** | 熬夜者的押金补贴早睡者的奖励，形成"好人有好报"的正反馈 |

### 1.4 目标用户画像

| 属性 | 描述 |
|------|------|
| **年龄层** | 18-35岁，大学生及年轻职场人 |
| **行为特征** | 经常熬夜刷手机/打游戏/追剧，明知有害但难以自控 |
| **社交场景** | 已有微信群（同学群、同事群、兴趣群），群内有互助氛围 |
| **心理需求** | 需要外部约束力帮自己改变，渴望被同伴认可 |
| **经济敏感度** | 10元押金门槛低但足以产生心理锚定效应 |

## 2. 核心功能需求

### 2.1 用户注册/登录流程

**采用微信静默授权 + 手机号确认的双步流程：**

```
步骤1：打开小程序 → 微信静默授权（获取 openid、昵称、头像）
步骤2：首次使用需绑定手机号（微信手机号快捷组件）
步骤3：选择要加入的早睡群（通过群分享链接/扫码加入）
步骤4：进入"缴押金"环节（未缴押金 = 游客身份，只能看排行榜）
```

**身份层级：**

| 身份 | 权限 |
|------|------|
| 游客（未缴押金） | 可查看排行榜、活动规则，不可打卡 |
| 正式成员（已缴押金） | 可打卡、查看排名、参与奖励分配 |
| 群主/管理员 | 可创建活动、调整规则、踢出成员 |

### 2.2 押金缴纳机制

#### 缴纳规则

| 项目 | 规则 |
|------|------|
| **押金金额** | 固定 10 元/人/轮 |
| **缴纳方式** | 微信支付（小程序内支付组件） |
| **缴纳时机** | 每轮活动开始前，必须在活动开始时缴纳 |
| **缴纳状态** | 缴纳后进入"正式成员"身份，状态实时可见 |

#### 退还规则

| 场景 | 退还逻辑 |
|------|----------|
| **周期正常结束** | 押金扣除罚金后，剩余部分自动退还至微信余额 |
| **中途退出** | 不退还押金，剩余押金并入本轮奖励池 |
| **全勤（7天全部合格打卡）** | 押金全额退还 + 经济奖励分成（罚金池充裕时）或荣誉表彰（罚金池不足时） |
| **群解散/活动取消** | 押金全额退还 |

> **设计原则**：押金不是"赌注"，而是"承诺金"。绝大部分人（只要打卡4天以上）都能拿回大部分押金，只有严重违约者才会被扣除较多。

### 2.3 打卡机制

#### 什么是"早睡"？—— 时间界定

**核心规则：在 21:00 ~ 23:00 之间完成打卡，即为"早睡成功"。**

| 时间段 | 定义 | 打卡结果 |
|--------|------|----------|
| **21:00 - 23:00** | 🟢 早睡黄金时段 | 合格打卡，获得 2 早睡币 |
| **23:00 - 23:30** | 🟡 勉强早睡 | 迟到打卡，获得 1 早睡币，扣 1 元押金 |
| **23:30 之后** | 🔴 熬夜了 | 未合格，获得 0 早睡币，扣 2 元押金 |
| **未打卡** | ⚫ 失联 | 默认熬夜，获得 0 早睡币，扣 2 元押金 |

#### 打卡方式

```
方式1：手动打卡
  → 进入打卡页 → 点击"我准备睡了"按钮 → 系统记录时间戳
  → 需要回答一个随机"睡前小问"（趣味互动 + 防作弊）

方式2：智能提醒打卡
  → 21:00 系统推送微信服务通知"该睡啦~"
  → 用户点击通知直接进入打卡页，一键完成
```

#### 睡前小问（防作弊 + 趣味性）

每次打卡弹出一道随机趣味题，5秒内点击正确选项即通过：
- "今天月亮是圆的还是弯的？"（常识题）
- "1+7等于几？"（简单算术）
- "你今天喝了几杯水？1杯/3杯/5杯"（无需验证的回忆题）

> 目的不是考知识，而是确认用户此刻确实在操作手机、有意准备睡觉。

### 2.5 七天活动周期

#### 周期结构

```
┌─────────────────────────────────────────────────────────────┐
│                    一轮活动 = 7 天                           │
│                                                             │
│  Day0        Day1     Day2     ...     Day7    Day8         │
│  报名缴费期   ┆打卡期┆  打卡期   ...    打卡期   结算日      │
│  (24h)      ┆       ┆                ┆       (24h)        │
│             └────────── 7天打卡 ──────────┆                │
└─────────────────────────────────────────────────────────────┘
```

| 阶段 | 时长 | 说明 |
|------|------|------|
| **报名缴费期** | 24小时 | 群主发起本轮活动，成员缴纳押金 |
| **打卡期** | 7天 | 每日21:00-23:30可打卡 |
| **结算日** | 24小时 | 系统自动计算排名、分配奖励、退还押金 |

#### 周期重置逻辑

- 结算完成后，自动进入下一轮的"报名缴费期"
- 上一轮的早睡币累积计入历史总分，但本轮排名归零重新计
- 成员可选择"休息一轮"不参与，不影响历史记录
- 群主可设定群的活动频率（每周一轮 / 每两周一轮）

| 维度 | 说明 |

| **群内排行** | 当前群所有成员的本轮排名 |
| **历史总榜** | 跨所有周期的累积早睡币排名 |
| **连续天数榜** | 当前最长连续早睡天数排名 |

#### 展示方式

- Top 3 显示头像 + 昵称 + 早睡币 + 称号图标
- 4-10名显示简要信息
- 自己的位置高亮标注
- 每日打卡状态用🟢🟡🔴⚫标记，形成"一周睡眠热力图"

#### 历史记录

- 每轮结算后生成"睡眠周报"卡片（可分享到群/朋友圈）
- 包含：本周打卡天数、早睡币、排名、连续天数、对比上周变化

## 3. 奖励规则设计（核心激励机制）

> **设计哲学**：押金不是赌局，而是承诺。奖励不是赢钱，而是回报。系统要让"好人有好报"——早睡的人拿回押金还能赚一点，熬夜的人付出代价但不至于血本无归。

### 3.1 奖励来源：押金池分配逻辑

```
一轮活动押金池 = 所有成员押金总和 = N × 10 元

押金池分为两部分：
┌──────────────────────────────────────┐
│           押金池 (N × 10元)          │
│                                      │
│  ┌────────────┐  ┌────────────────┐  │
│  │ 个人押金区  │  │  罚金池(公共)  │  │
│  │ (每人10元) │  │  (扣罚汇集)   │  │
│  └────────────┘  ┌────────────────┘  │
│                                      │
│  个人押金区：结算后退还剩余部分        │
│  罚金池：分配给早睡表现好的成员        │
└──────────────────────────────────────┘
```

**核心逻辑**：

- 每人押金10元，属于"个人押金区"
- 每次未合格打卡，从个人押金中扣除罚金，罚金进入"罚金池"
- 结算时：个人押金区的剩余退还本人；罚金池分配给合格成员

### 3.2 罚金扣除规则（逐日累计）

| 打卡表现 | 每日罚金 | 说明 |
|----------|---------|------|
| 🟢 早睡黄金时段 | **0 元** | 不扣罚 |
| 🟡 勉强早睡 (23:00-23:30) | **1 元** | 轻罚，鼓励尽量在黄金时段 |
| 🔴 熬夜 (23:30后打卡) | **2 元** | 重罚 |
| ⚫ 未打卡 | **2 元** | 同熬夜罚 |

**7天最坏情况**：7天全未打卡 = 扣14元 > 押金10元 → **封顶扣除10元**（押金全部归入罚金池，不再多扣）

> 封顶设计确保最差表现也只是损失10元押金，不会产生负债。

### 3.3 奖励分级设计

#### 分配公式

```
罚金池总额 P = Σ(所有成员被扣除的罚金)

奖励分配权重 W_i = 基础权重 + 全勤加成

基础权重 = 成员i的本轮早睡币数量
全勤加成 = 如果7天全部🟢合格打卡 → 基础权重 × 1.5（即加成50%）

成员i的奖励 = P × (W_i / ΣW_all)
```

#### 奖励分级表

| 打卡表现 | 早睡币 | 权重 | 押金退还 | 预期奖励 | 总收益 |
|----------|--------|------|---------|---------|--------|
| 🏆 全勤早睡(7天🟢) | 20 | 20×1.5=30 | 10元全额 | 较高份额或荣誉表彰 | ≥10元 + 荣誉激励 |
| ⭐ 全勤(含🟡) | 16-18 | 16-18 | 7-9元 | 中等份额 | ≈10元 |
| 👍 合格4-6天 | 8-14 | 8-14 | 4-8元 | 较低份额 | <10元 |
| 😐 合格1-3天 | 2-6 | 2-6 | 2-6元 | 极低份额 | <6元 |
| 💀 0天合格 | 0 | 0 | 0元 | 0 | 0元 |

### 3.4 计算示例（10人群）

```
10人参与，每人押金10元，总押金池100元

打卡结果假设：
- 3人全勤早睡(7天🟢)：每人扣0元，早睡币20，权重30
- 2人全勤(含🟡)：每人扣1-2元，早睡币16-17，权重16-17
- 3人合格4-6天：每人扣3-6元，早睡币8-12，权重8-12
- 2人几乎未打卡：每人扣10元(封顶)，早睡币0，权重0

罚金池 P = 3×0 + 2×2 + 3×5 + 2×10 = 0+4+15+20 = 39元

总权重 ΣW = 3×30 + 2×17 + 3×10 + 2×0 = 90+34+30+0 = 154

全勤早睡者奖励 = 39 × (30/154) ≈ 7.6元
  → 退还10元 + 奖励7.6元 = 总收入17.6元 ✅ 丰厚回报

全勤含🟡者奖励 = 39 × (17/154) ≈ 4.3元
  → 退还8元 + 奖励4.3元 = 总收入12.3元 ✅ 有回报

合格4-6天者奖励 = 39 × (10/154) ≈ 2.5元
  → 退还5元 + 奖励2.5元 = 总收入7.5元 ⚠️ 小亏，但可接受

几乎未打卡者奖励 = 0元
  → 退还0元 + 奖励0元 = 总收入0元 ❌ 全损，但仅10元
```

**结论**：全勤者净赚约7.6元，偶尔失败者小亏2.5元，严重失败者损失10元。激励梯度合理。

### 3.5 特殊奖励机制

#### 🎯 "早睡之星"额外红包

- 每轮排行榜第1名，额外获得**"早睡之星红包"** = 罚金池的 10%
- 这是从罚金池中先抽取10%作为冠军独享，剩余90%再按权重分配

#### 🤝 "拯救奖励"（人情味设计）

- 如果某成员因特殊原因（生病、加班等）申请"请假"1天：
  - 群主可批准"请假"，该天按🟡处理（扣1元而非2元）
  - 每轮最多请假1天，需在当天23:30前申请
  - 防止滥用：请假天数不计入连续早睡奖励

#### 🏅 "全勤荣耀"机制（非经济激励增强）

> **设计背景**：当全勤率高、罚金池金额小时，经济激励力度不足。此时通过社交荣誉激励弥补，让"坚持早睡"本身成为值得骄傲的事。经济激励与社交荣誉双轨并行，确保无论全勤率高低，参与者始终有正向回报。

**触发条件**：罚金池人均分配额 < 2元（即大多数人表现良好时），非经济激励权重自动提升

| 激励层级 | 内容 | 触发条件 |
|----------|------|---------|
| **🏆 群内公告表彰** | 结算日自动在群内推送全勤名单，附专属称号和排名 | 全勤者 |
| **🎖 本轮专属勋章** | 每轮生成独特勋章（如"第3轮🌙守夜卫士"），展示在个人主页勋章墙 | 全勤者 |
| **🌟 荣耀轮标记** | 本轮全勤率 ≥ 80% 时，该轮标记为"荣耀轮"，群排行榜永久展示 | 全群 |
| **📸 全勤合照卡片** | 系统生成全勤成员头像合成的"早睡战队"卡片，可分享到群/朋友圈 | 全勤者 |
| **🔥 连续全勤里程碑** | 连续全勤2轮→解锁"铁人"称号；3轮→"战神"；5轮→"传说"（跨轮累积） | 跨轮累积 |

**非经济激励 vs 经济激励对比**：

| 场景 | 经济激励 | 非经济激励 |
|------|---------|-----------|
| 正常混合（有人熬夜） | ✅ 罚金池充足，奖励有吸引力 | ✅ 称号/勋章同步生效 |
| 大多数人全勤 | ⚠️ 罚金池小，奖励微薄 | ✅ 荣耀机制重点发力，社交荣誉感增强 |
| 所有人全勤 | ❌ 罚金池为零，无经济奖励 | ✅ 全面荣耀表彰 + 专属"全员荣耀轮"庆典 |

> **核心思路**：经济激励解决"要不要坚持"的问题，非经济激励解决"坚持了有没有意义"的问题。两者互补，确保无论全勤率高低，参与者始终有正向回报。

#### ⚖️ 边界情况兜底规则

| 边界情况 | 处理方式 |
|----------|---------|
| **所有人全勤（罚金池 P = 0）** | 押金全额退还每人10元 + 全员获得"全员荣耀"称号 + 生成全员庆祝卡片 |
| **所有人失败（ΣW_all = 0）** | 罚金池均等退还给所有成员（每人10元），不产生无人领取的悬空资金 |
| **仅1人合格，其余全失败** | 合格者获得冠军红包 + 剩余罚金池全额奖励；其余人押金归入罚金池 |
| **罚金池人均分配额 < 1元** | 奖励累积到下一轮合并发放（最低提现门槛1元）；本轮以非经济激励为主 |

### 3.6 奖励发放方式

| 项目 | 规则 |
|------|------|
| **发放时机** | 周期结算日（Day8）自动计算 |
| **发放方式** | 微信余额提现（小程序提现接口） |
| **最低提现门槛** | 1元（低于1元不提现，累积到下轮） |
| **提现手续费** | 0（由平台承担，增强用户体验） |
| **退还押金** | 同样通过微信余额自动退还 |
| **发放时效** | 结算后24小时内完成 |

### 3.7 防作弊机制

| 风险 | 对策 |
|------|------|
| **代打卡** | 睡前小问验证 + 打卡需在手机端操作（不能自动化脚本） |
| **改时间** | 使用服务器时间戳，不信任客户端时间 |
| **假装睡觉** | 打卡后如果微信活跃度检测（如有新消息读取），标记可疑 |
| **多人串通** | 罚金池按权重分配，串通无意义——总池固定 |
| **群主作弊** | 群主也参与打卡，与其他成员同规则，无特殊豁免 |

### 3.8 奖励规则总结

```
一句话总结：
"经济激励与社交荣誉双轮驱动——罚金池充裕时早睡者赚钱，
 罚金池不足时早睡者赚荣誉，无论哪种场景坚持早睡都有回报。"

核心数学：
  罚金池 = Σ(每日罚金) → 按早睡币权重分配给合格成员
  全勤加成 = 权重 ×1.5
  冠军红包 = 罚金池 ×10%
  罚金封顶 = 10元/人/轮（等于押金，不产生负债）

激励双轨：
  经济轨：罚金池 → 权重分配 → 微信余额发放
  荣誉轨：称号 + 勋章 + 群公告 + 合照卡片 + 连续里程碑
  触发切换：罚金池人均 < 2元 → 荣誉轨权重提升

边界兜底：
  P = 0 → 全员押金退还 + 全员荣耀表彰
  ΣW = 0 → 罚金池均等退还所有成员
```

## 7. 风险与约束

### 7.1 微信支付合规性

| 风险 | 级别 | 对策 |
|------|------|------|
| **押金模式审核风险** | 🔴 高 | 微信可能将押金模式归类为"赌博/竞猜"而拒绝审核。对策：在提交审核时强调这是"自律工具"，押金是承诺金而非赌注，奖励来源于违约者而非平台 |
| **提现功能合规** | 🟡 中 | 商户付款到零钱需企业资质。对策：注册企业主体，申请微信商户号，确保资金链路合规 |
| **资金池合规** | 🟡 中 | 罚金池本质是成员间资金流转。对策：平台不截留任何资金，所有罚金100%分配给成员，平台仅做通道 |

### 7.2 押金退还政策

| 风险 | 级别 | 对策 |
|------|------|------|
| **退款纠纷** | 🟡 中 | 明确规则写在用户协议中：押金扣除基于客观时间戳，不可争议 |
| **中途退出不退** | 🟡 中 | 退出时二次确认弹窗，明确告知"退出后押金不退"，减少投诉 |
| **群解散退款** | 🟢 低 | 群解散时自动全额退款，无争议空间 |

### 7.3 用户流失应对

| 风险 | 级别 | 对策 |
|------|------|------|
| **首轮后流失** | 🔴 高 | 新用户首轮给予"新手保护"：1天未打卡不扣罚金（给适应期） |
| **连续失败流失** | 🟡 中 | 失败后推送鼓励消息"这周不太好，下周再来！" + 下一轮押金可打8折 |
| **群冷场** | 🟡 中 | 群活跃度低时自动推送"本周已有X人打卡，你还不来？" |
| **无聊感** | 🟡 中 | 称号系统 + 周报卡片 + 限时挑战活动保持新鲜感 |

### 7.4 其他风险

| 风险 | 级别 | 对策 |
|------|------|------|
| **时间篡改作弊** | 🟡 中 | 服务器时间戳为唯一标准 |
| **代打卡作弊** | 🟢 低 | 睡前小问 + 手机端操作要求 |
| **群主权力滥用** | 🟢 低 | 群主同规则参与，请假审批公开可见 |
| **微信审核被拒** | 🔴 高 | 需准备完整合规材料，可能需要调整产品定位描述 |
